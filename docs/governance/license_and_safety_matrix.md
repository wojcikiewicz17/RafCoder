# Matriz de Licença, Segurança Informacional e Uso Responsável

> **Aviso:** este documento é técnico e operacional, não substitui aconselhamento jurídico.

## 1) O que a licença permite (resumo objetivo)

### Código do repositório (`LICENSE-CODE`)
- Licença: **MIT**.
- Permite uso, modificação, distribuição e **venda** de software.
- Exige manter aviso de copyright/licença.

### Modelo e derivados (`LICENSE-MODEL`)
- Licença própria DeepSeek com permissões amplas de uso/distribuição.
- **Não define regra geral de “somente não comercial”**.
- Exige manter restrições de uso do Attachment A em redistribuição de modelo/derivados.
- Proíbe usos como: dano a menores, discriminação, desinformação para causar dano, uso militar, etc.

## 2) Conclusão de coerência com seu pedido

Seu pedido de:
- uso não comercial,
- uso somente científico/estudos,
- proteção humana,
- proteção de crianças,
- inclusão social global,

é **coerente como política adicional de projeto/organização**.

Mas isso deve ser tratado como:
1. **Política interna adicional** (não como “o que a licença MIT obriga”).
2. **Termo de uso para distribuição/serviço** quando você hospedar API/app.
3. **Camada de governança** com controles técnicos e auditoria.

## 3) Separação recomendada de documentos

Para operacionalizar, separar em três documentos:

1. `docs/governance/license_and_safety_matrix.md` (este arquivo)
   - mapeia obrigação de licença vs política adicional.
2. `docs/governance/research_only_and_noncommercial_policy.md`
   - define política de uso restrito (não comercial + pesquisa).
3. `docs/governance/child_protection_and_global_inclusion_standard.md`
   - define controles de proteção infantil, direitos humanos e inclusão.
